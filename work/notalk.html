Overview

프로젝트 개요

배경
스마트폰 보급률이 높아졌지만 노인 사용자의 디지털 소외는 여전히 심각합니다.
문제
기존 메시지 앱은 작은 글씨, 복잡한 UI로 노인 사용자가 혼자 사용하기 어렵습니다.
목표
누군가의 도움 없이 가족과 연락할 수 있는, 직관적인 메시지 앱을 설계합니다.
결과
단순하고 큰 UI, 음성 입력, 자주 쓰는 메시지 등 노인 친화적 기능 중심으로 설계.

Problem Definition

핵심 문제

01

가독성 부족

기존 앱의 작은 글씨와 낮은 대비, 복잡한 아이콘은 시력이 저하된 노인 사용자에게 큰 장벽입니다. 확대 기능이 있어도 레이아웃이 깨지는 경우가 많습니다.

02

사용자 흐름 복잡성

메시지 전송까지 3–5단계의 탭이 필요하고, 실수로 뒤로 가거나 다른 화면으로 이동했을 때 다시 돌아오는 방법을 모르는 경우가 많습니다.

03

입력 방식의 한계

손 떨림이나 관절 문제로 터치 정확도가 낮고, 작은 키보드 타이핑이 어려운 사용자에게 텍스트 입력 자체가 큰 부담입니다.

User Needs

사용자 니즈

👁️

잘 보이는 화면

큰 글씨, 높은 대비, 충분한 여백. 시력이 낮아도 화면 내용을 한눈에 파악할 수 있어야 합니다.

👆

쉬운 조작

큰 터치 영역, 단순한 탐색 구조. 실수로 잘못 눌러도 쉽게 되돌아올 수 있어야 합니다.

🎤

편한 입력

음성 입력, 자주 쓰는 메시지 저장, 큰 키보드. 타이핑이 힘들어도 메시지를 보낼 수 있어야 합니다.

Project Goals

설계 목표

01

3탭 이내로 메시지 전송 완료

최소한의 단계로 핵심 기능에 도달할 수 있는 간결한 User Flow

02

AA 이상 대비율 · 최소 18px 이상 텍스트

WCAG 접근성 기준을 충족하는 시각적 설계

03

음성 입력 및 즐겨찾기 메시지 기능 제공

타이핑이 어려운 사용자를 위한 대안적 입력 방식

04

명확한 피드백 — 전송 완료, 오류 상태 표시

불안감을 줄이는 시스템 상태 가시성 확보

05

구조적으로 복잡하지 않은 단일 목적 앱

NOTALK는 오직 가족과의 연락에만 집중합니다

🎯 배운 점

접근성은 특정 사용자를 위한 부가 기능이 아니라, 모든 사용자 경험의 기준이 되어야 한다는 것을 배웠습니다. 단순함이 얼마나 어려운 설계 결정인지도 실감했습니다.

🔍 아쉬운 점

실제 노인 사용자를 대상으로 한 직접 인터뷰와 사용성 테스트를 진행하지 못했습니다. 다음 프로젝트에서는 반드시 실사용자 검증 단계를 포함하고 싶습니다.
